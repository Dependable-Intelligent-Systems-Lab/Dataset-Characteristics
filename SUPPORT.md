# Support

Use GitHub issues for reproducible bugs, feature requests, packaging problems, documentation fixes, and dataset characteristic proposals.

Before opening an issue:

- Check the README and existing issues.
- Prepare a minimal reproducible example when reporting a bug.
- Include Python, package, pandas, and scikit-learn versions for runtime issues.
- Include dataset source and license information for data-related questions.

Security vulnerabilities and sensitive data concerns should be reported privately through [SECURITY.md](SECURITY.md).
